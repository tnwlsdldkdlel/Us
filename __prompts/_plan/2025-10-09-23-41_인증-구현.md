## Plan
1. 기존 앱 구조와 상태 관리를 파악해 인증 게이트 진입점을 정의한다.
2. Supabase SDK 초기화 및 인증 저장소/유저 저장소를 구현해 로그인 후 사용자 레코드를 보장한다.
3. Google/Apple 로그인 UI를 구성하고 오류/로딩 처리, 홈 화면 전환을 연결한다.
4. 설정 화면에 로그아웃 동작과 사용자 정보 표시를 추가한다.
5. 정적 분석으로 컴파일 오류 여부를 확인하고 필요한 후속 조치를 정리한다.

## Working
- `Supabase.initialize` 구성, `AuthGate`로 세션 감지 및 사용자 레코드 upsert 처리
- `LoginScreen`에 Google/Apple 버튼 추가, 예외 처리 및 로딩 상태 관리
- `SettingsScreen`에서 Supabase 세션 정보를 활용해 프로필 표시/로그아웃 처리
- `flutter analyze` 실행 시 기존 코드의 `withOpacity` 경고 다수 확인 (현 시점 미해결)

## Result
- Day1 범위의 인증 플로우 구현 완료 (`lib/main.dart`, `lib/app/auth_gate.dart`, `lib/screens/auth/login_screen.dart`, `lib/screens/settings/screens/settings_screen.dart`, `lib/data/auth/auth_repository.dart`, `lib/data/user/user_repository.dart`, `lib/config/supabase_config.dart`)
- Supabase OAuth 로그인/로그아웃 및 사용자 프로필 동기화 로직 추가, 홈 진입 라우팅과 연계 완료
- 정적 분석에서 기존 경고(`withOpacity` 사용)만 남아 있으며 신규 오류는 없음
